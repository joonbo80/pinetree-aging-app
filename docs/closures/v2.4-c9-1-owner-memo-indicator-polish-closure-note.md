# v2.4 C9.1 Owner/Memo Indicator Polish - Closure Note

Date: 2026-05-15
Status: CLOSED
Parent: `docs/closures/v2.4-c9-owner-memo-lite-ui-closure-note.md`

## 1. Status

C9.1 Owner/Memo Indicator Polish is closed.

## 2. Purpose

C9 saved Owner/Memo metadata successfully, but collapsed party rows did
not show whether workflow metadata existed.

C9.1 adds a lightweight scanning layer so collectors can see Owner/Memo
presence from the party rollup list.

## 3. Implementation Summary

### 3.1 API

Added:

```text
GET /api/workflow/items?workspaceId=...
```

The endpoint returns workflow list items for the current workspace using
the C8 SharePoint field-name mapping pattern.

### 3.2 UI

`PartyRollupTable` now:

- Loads workflow summaries for the active workspace.
- Stores summaries by `workflowKey`.
- Shows `Owner: {name}` and `Memo` badges in the Party column.
- Updates badges immediately after saving from the expanded panel.

## 4. UX Rule

Collapsed rows show only presence indicators.

The memo body remains inside the expanded Owner/Memo panel to keep the
Workbench calm and scan-friendly.

## 5. Known Boundaries

C9.1 does not yet solve:

- Workflow list pagination beyond the first page.
- Filtering by memo/owner.
- Owner picker from tenant users.
- Memo history.
- Promise tracking.

Those remain future tasks.
