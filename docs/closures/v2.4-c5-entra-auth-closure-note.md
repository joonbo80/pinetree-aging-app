```text
Audience: 00000003-0000-0000-c000-000000000000

```

Implementation note:

C5 originally attempted local Microsoft JWT signature validation. That
was too much surface area for a spike and hit Microsoft delegated token
signing-key edge cases. The accepted C5 approach is:

1. Perform local tenant, issuer, audience, expiration, and scope checks.
2. Call Microsoft Graph `/me`.
3. Accept the delegated token when Graph accepts it and the returned
   user id matches the token object id.

Full production JWT validation can be revisited later if needed.

## 6. Standing Rules Learned

### 6.1 Do Not Confuse Admin With Delegated Site Access

Delegated Graph tokens act as the signed-in user. M365 admin rights do
not automatically grant delegated Graph access to every Teams private
channel SharePoint site.

Defense: verify private-channel membership for the actual signed-in
user before treating SharePoint Graph 403s as code bugs.

### 6.2 Prefer Direct Site ID for Private Channel Sites

Path lookup and search can fail or return restricted results for private
channel sites.

Defense: pin the Graph site ID:

```text
pinetreeexpress.sharepoint.com,0b39ac5a-ed5c-40f7-bc40-60c30bd936b6,0fc32f15-b8b1-4c6e-9854-2eb04a096f04
```

### 6.3 Keep C5 Narrow

C5 proved auth and a SharePoint write/delete round-trip. It did not
implement production snapshot upload, workflow metadata, sharing, or
multi-user editing.
