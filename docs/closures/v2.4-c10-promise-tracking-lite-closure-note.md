# v2.4 C10 Promise Tracking Lite - Closure Note

Date: 2026-05-15
Status: CLOSED
Parent roadmap: `docs/specs/v2.4-phase3-persistence-roadmap.md`

## 1. Status

C10 Promise Tracking Lite is closed.

## 2. Purpose

C9 gave collectors a SharePoint-backed place to record Owner/Memo
context. C9.1 made that context visible from collapsed party rows.

C10 adds the next lightweight workflow layer:

```text
Promise Date
Promise Amount
Promise Status
```

This lets the Statement Collection Workbench record a simple payment
commitment without turning the app into a payment system or CRM.

## 3. Implementation Summary

### 3.1 API

The workflow item API now reads and writes:

```text
promiseDate
promiseAmount
promiseStatus
```

The existing SharePoint display-name to internal-name mapping remains in
place.

### 3.2 UI

The expanded workflow panel now contains:

- Owner.
- Promise Date.
- Promise Amount.
- Promise Status.
- Memo.

Collapsed rows show a `Promise` badge when any promise field exists.

## 4. Data Rules

Promise values remain scoped by the existing workflow key:

```text
{workspaceId}__{partyKey}__{currency}__{direction}
```

This preserves:

- Party separation.
- Currency separation.
- AR/AP direction separation.

## 5. UX Rule

C10 remains Lite.

The row badge shows that a promise exists, but the detailed promise data
stays inside the expanded row panel. This keeps the Workbench calm and
scan-friendly.

## 6. Verification

Build verification:

```text
aging-api build: PASS
aging-app build: PASS
```

Expected browser verification:

1. Sign in on `/spike/auth`.
2. Open `/aging`.
3. Expand a party row.
4. Enter Promise Date, Promise Amount, and Promise Status.
5. Save.
6. Confirm `Saved to SharePoint.`
7. Collapse the row and confirm a `Promise` badge appears.
8. Reload and confirm the promise values return from SharePoint.

## 7. Non-goals Confirmed

C10 does not include:

- Email sending.
- Outlook draft generation.
- Payment automation.
- Reminder automation.
- Promise history.
- Multi-user conflict handling.

## 8. Recommended Next Step

Recommended next step:

```text
C11 Email Draft Lite
```

C11 should generate a safe email draft from party, balance, memo, and
promise context. Automatic sending should remain out of scope until
Mail.Send permissions, review UX, and audit expectations are settled.
