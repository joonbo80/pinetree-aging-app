## 1. Status

C8 Workflow Metadata List Integration is closed.

C8 proved that `aging-api` can use Microsoft Graph delegated auth to
create, read, update, and delete workflow metadata rows in the
SharePoint list:

```text
PE Aging&Statement Workflow
```

## 4. Implemented Spike

Endpoint:

```text
POST /api/spike/workflow-roundtrip
Authorization: Bearer <Microsoft Graph delegated token>
```

The route:

1. Validates the delegated Microsoft Graph token.
2. Resolves the SharePoint site.
3. Resolves the workflow list by display name.
4. Loads SharePoint list columns.
5. Maps display names to internal field names.
6. Creates a test item.
7. Reads the created item.
8. Updates `memoText` and `promiseStatus`.
9. Reads the updated item.
10. Deletes the test item.
11. Returns cleanup status.

## 6. Important Finding

Initial C8 test failed with:

```text
Field 'direction' is not recognized
```

Root cause:

SharePoint column display names are not always the field names accepted
by Microsoft Graph list item writes.

Fix:

The C8 route now reads list columns first and builds a display-name to
internal-name map before creating or updating list items.

This rule should be preserved for all future SharePoint list
integration work.

## 7. Standing Rules Preserved

C8 preserves:

- Microsoft Entra delegated auth from C5.
- SharePoint target site from C5/C6.
- Browser-local fallback from C3.
- Snapshot read/write flow from C6/C7.
- AR/AP and USD/CAD separation in workflow keys.
- No production workflow UI yet.
- No multi-user edit or conflict handling.

## 8. Known Boundaries

C8 does not solve:

- Owner assignment UI.
- Memo editing UI.
- Promise tracking UI.
- Metadata row merge rules.
- Conflict resolution.
- List row history/audit UI.

Those are future tasks.
