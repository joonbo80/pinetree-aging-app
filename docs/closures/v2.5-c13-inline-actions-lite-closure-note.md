## 1. Status

C13 Inline Actions Lite is closed.

C13 adds the first write-enabled action panel to the v2.5 Workflow
Queue:

```text
/workflow row
  -> Actions
  -> Inline Actions Lite
  -> Quick memo + Promise status
  -> SharePoint workflow row update
```

The scope is intentionally small. C13 does not add bulk update,
automatic email sending, reminders, notifications, or multi-user
conflict handling.

## 2. Purpose

C12 proved that the Workflow Queue can show SharePoint-backed workflow
metadata as a daily collector surface.

C13 proves the next smallest write path: a collector can update a row's
memo and promise status directly from the queue without opening Party
Detail or the full Workbench panel.

The goal is small workflow bookkeeping, not automation.

## 3. Implementation Summary

### 3.1 Inline Panel

`WorkflowQueuePage` now includes a row-level action surface:

```text
Actions
  -> Inline Actions Lite
```

The panel includes:

- Quick memo textarea.
- Promise status select.
- Save inline updates button.
- Per-row saving / success / error message.

### 3.2 Write Path

C13 reuses the existing v2.4 workflow write path:

```text
apiClient.writeWorkflowItem(...)
POST /api/workflow/item
PE Aging&Statement Workflow SharePoint list
```

No new SharePoint list is created.
No new SharePoint columns are required.

### 3.3 Minimum Scope

Implemented minimum scope:

```text
Quick memo save
Promise status update
```

Deferred:

```text
Mark contacted
Snooze / next follow-up date
Owner picker
Bulk update
Email send
Notifications
```

## 4. Data Safety

C13 writes to one workflow row at a time using the existing workflow key:

```text
workspaceId + partyKey + currency + direction
```

This preserves:

- USD/CAD separation.
- AR/AP separation.
- Party/currency/direction row identity.

The update payload keeps existing owner, promise date, promise amount,
party, currency, and direction values unless the inline panel explicitly
changes memo or promise status.

## 5. Error Handling

If a save fails:

- The row remains expanded.
- The typed memo/status draft remains visible.
- The page shows an inline error.
- The UI does not pretend the save succeeded.

If a save succeeds:

- The returned SharePoint workflow row is merged into local queue state.
- The row displays `Saved to SharePoint.`

## 8. Standing Rules Maintained

C13 maintains the standing rules:

- USD and CAD are not merged.
- AR and AP are not netted.
- Workflow writes are scoped to one workflow key.
- SharePoint metadata remains additive to the committed snapshot.
- Browser-local snapshot fallback remains valid.
- Email drafts are not sent automatically.
- Calm visual tone remains the default.

## 9. Known Boundaries

C13 does not solve:

- Owner identity normalization.
- Owner picker.
- Mark contacted timestamp.
- Snooze-specific field.
- Action history.
- Conflict resolution.
- Multi-user edit locking.
- Bulk edit.
- Notifications.
- Email sending through Graph.

These remain future work.
