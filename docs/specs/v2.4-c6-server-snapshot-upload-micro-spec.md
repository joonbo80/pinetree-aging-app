## 2. Goal

Upload the current C3 snapshot envelope to SharePoint as JSON.

The upload is user-triggered, explicit, and additive. Browser-local
snapshot persistence remains the primary fallback.

## 3. Non-goals

C6 does not include:

- Server restore/download.
- Snapshot listing.
- Workflow metadata list writes.
- Owner / memo / promise UI.
- Multi-user edit.
- Conflict resolution.
- Replacing browser-local C3 restore.
- Production auth rollout for every route.

## 6. File Naming

C6 uses the existing C3 filename pattern:

```text
aging-snapshot-{asOfDate}-{importBatchId}.json
```

If the same filename already exists, C6 overwrites it. This is
acceptable for C6 because the snapshot represents the current committed
workspace for the same import batch.

Future C7/C8 can add version listing or immutable historical copies.

## 7. API Contract

Endpoint:

```text
POST /api/snapshot/upload
Authorization: Bearer <Microsoft Graph delegated token>
Content-Type: application/json
```

Request body:

```json
{
  "snapshot": { "...": "AgingSnapshotV1" },
  "filename": "aging-snapshot-2026-05-01-batch-id.json"
}
```

Response body:

```json
{
  "status": "ok",
  "source": "sharepoint-snapshot-upload",
  "filename": "aging-snapshot-2026-05-01-batch-id.json",
  "folder": "AGING&STATEMENT APP",
  "site": {
    "id": "...",
    "displayName": "PINETREE-ACCOUNTING",
    "webUrl": "..."
  },
  "file": {
    "id": "...",
    "name": "...",
    "size": 123456,
    "webUrl": "..."
  }
}
```

## 8. Validation

The API validates before uploading:

- Bearer token present.
- Token accepted by C5 delegated Graph validation.
- Snapshot body is an object.
- `snapshot.snapshotVersion === "v2.3-c3-snapshot-v1"`.
- `snapshot.summary.importBatchId` exists.
- `snapshot.summary.asOfDate` exists.
- `snapshot.result.uploadSession` exists.
- Filename ends with `.json`.
- Filename does not contain `/`, `\`, or `..`.

Validation failures return `400`.
Auth failures return `401`.
Graph failures return `500`.

## 9. Frontend UX

C6 adds a dev/spike upload control to `/spike/auth`.

Reason: C6 is still Phase 3 plumbing. The main workbench UI should not
show server upload until C6 is closed and C7 restore/listing is designed.

The `/spike/auth` page should:

- Use the current signed-in token.
- Load the current browser-local snapshot from C3 storage.
- Upload it to SharePoint.
- Show filename, size, SharePoint web URL, and status.

If no browser-local snapshot exists, the page should show a clear
message instructing J to load baseline and confirm import first.
