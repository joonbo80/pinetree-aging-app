# v2.4 C10 Promise Tracking Lite Micro-spec

Date: 2026-05-15
Status: CLOSED
Parent: `docs/specs/v2.4-c9-owner-memo-lite-ui-micro-spec.md`
Previous step: `docs/closures/v2.4-c9-1-owner-memo-indicator-polish-closure-note.md`

## 1. Goal

C10 extends the SharePoint-backed Owner/Memo workflow panel with a
minimal promise tracking surface.

The purpose is to capture a collector's lightweight commitment record:

```text
Promise Date
Promise Amount
Promise Status
```

## 2. Scope

C10 adds:

- Promise Date input.
- Promise Amount input.
- Promise Status select.
- SharePoint save/read for promise fields.
- Collapsed row `Promise` badge.

The existing Owner/Memo fields remain in the same panel.

## 3. SharePoint Fields

C10 uses the workflow list columns created during SharePoint discovery:

```text
promiseDate
promiseAmount
promiseStatus
```

The browser date input uses ISO format (`YYYY-MM-DD`). SharePoint may
display the same value as `MM/DD/YYYY` in the list UI.

## 4. API

Existing endpoints are extended:

```text
GET  /api/workflow/item?workflowKey=...
GET  /api/workflow/items?workspaceId=...
POST /api/workflow/item
```

All endpoints continue to require Microsoft Graph delegated token auth.

## 5. UI

The expanded party row panel is renamed conceptually to:

```text
Owner / Memo / Promise
```

Promise status options:

```text
None
Open
Follow Up
Paid
Broken
```

The collapsed row shows a compact `Promise` badge when any promise field
exists. Date is preferred for the badge label, then status, then amount.

## 6. Non-goals

C10 does not include:

- Email sending.
- Outlook draft generation.
- Reminder automation.
- Promise history.
- Conflict resolution.
- Calendar integration.
- Payment reconciliation.

## 7. Completion Criteria

C10 is complete when:

- API build passes.
- App build passes.
- Promise fields save to SharePoint.
- Promise fields reload from SharePoint.
- Collapsed rows show a Promise badge when promise data exists.
- Owner/Memo behavior remains unchanged.
