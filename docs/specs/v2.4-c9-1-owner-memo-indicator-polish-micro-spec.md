# v2.4 C9.1 Owner/Memo Indicator Polish Micro-spec

Date: 2026-05-15
Status: CLOSED
Parent: `docs/specs/v2.4-c9-owner-memo-lite-ui-micro-spec.md`

## 1. Goal

C9 proved that Owner/Memo values can be saved to the SharePoint
workflow list from expanded party rows.

C9.1 fixes the immediate workflow visibility gap:

```text
Collectors need to know which party rows already have Owner/Memo
metadata without opening every row one by one.
```

## 2. User Problem

After saving a memo for a party such as IATA CARGO, the memo existed in
SharePoint but the collapsed party rollup list did not show that any
memo existed.

That made the list harder to scan and created unnecessary re-opening of
rows.

## 3. Scope

C9.1 adds lightweight row indicators:

- `Owner: {name}` badge when an owner exists.
- `Memo` badge when memo text exists.
- Badges appear inside the Party column under the party key/signals.
- The memo body is not shown in the collapsed row.

## 4. Server Read Path

New endpoint:

```text
GET /api/workflow/items?workspaceId=...
```

The endpoint:

- Requires the existing Microsoft Graph delegated token.
- Resolves the SharePoint workflow list.
- Reads workflow list items.
- Returns items for the current `workspaceId`.
- Uses SharePoint display-name to internal-name mapping from C8.

This is intentionally simple and bounded for C9.1. It currently reads
the first workflow list page and filters by workspace.

## 5. UI Read Path

`PartyRollupTable` now loads workflow summaries for the active
workspace when the table mounts.

The table keeps a local summary map:

```text
workflowKey -> ownerDisplayName / memoText
```

When a row is expanded, the existing C9 panel still reads the exact item.
When the row is saved, the parent table summary is updated immediately.

## 6. Non-goals

C9.1 does not include:

- Memo body preview in collapsed rows.
- Owner picker.
- Bulk workflow prefetch pagination.
- Promise tracking.
- Conflict detection.
- Workflow metadata filter chips.

## 7. Completion Criteria

C9.1 is complete when:

- API build passes.
- App build passes.
- Saved Owner/Memo values can appear as row badges.
- Reloaded workspace can read existing workflow items and show row
  badges.
- Existing C9 expanded Owner/Memo save flow still works.
