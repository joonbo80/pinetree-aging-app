## 2. Goal

Download the latest uploaded C3 snapshot JSON from SharePoint and
restore it into the app using the existing C3 snapshot parser/storage
logic.

## 3. Non-goals

C7 does not include:

- Snapshot version picker.
- Snapshot delete UI.
- Snapshot diff.
- Workflow metadata list integration.
- Owner / memo / promise tracking.
- Multi-user edit.
- Conflict resolution.

## 4. API Contract

Endpoint:

```text
GET /api/snapshot/latest
Authorization: Bearer <Microsoft Graph delegated token>
```

Response:

```json
{
  "status": "ok",
  "source": "sharepoint-snapshot-latest",
  "filename": "aging-snapshot-2026-05-01-batch.json",
  "folder": "AGING&STATEMENT APP",
  "file": {
    "id": "...",
    "name": "...",
    "size": 1798433,
    "webUrl": "..."
  },
  "snapshot": {
    "snapshotVersion": "v2.3-c3-snapshot-v1"
  }
}
```

If no snapshot exists, return `404`.

## 5. Latest Selection Rule

C7 chooses the latest server snapshot by:

1. Folder children under `AGING&STATEMENT APP`.
2. Filename starts with `aging-snapshot-`.
3. Filename ends with `.json`.
4. Sort by SharePoint `lastModifiedDateTime` descending.
5. Download the first item.

This is intentionally simple. C8+ can add version UI or immutable
history rules.

## 6. Frontend UX

Add to `/spike/auth`:

```text
Restore latest server snapshot
```

Behavior:

- Requires signed-in token.
- Calls `GET /api/snapshot/latest`.
- Parses the snapshot through existing `parseSnapshotText`.
- Saves it to browser localStorage through `saveSnapshotToStorage`.
- Displays summary.
- Does not automatically navigate away from `/spike/auth`.

J can then hard-load `/aging` to verify restored workspace behavior.

## 8. Standing Rules

C7 preserves:

- C3 snapshot envelope shape.
- Browser-local fallback.
- C5 direct SharePoint site ID.
- C6 upload file naming.
- No workflow metadata writes.
- No multi-user edit.
