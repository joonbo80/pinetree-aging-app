## 2. Goal

Prove create/read/update/delete access for workflow metadata rows.

This is the server-side foundation for future Owner/Memo/Promise UI.

## 3. Non-goals

C8 does not include:

- Production Owner/Memo UI.
- Promise tracking UI.
- Multi-user editing.
- Conflict resolution.
- Row history display.
- Workflow metadata merge rules.

## 5. Spike Contract

Endpoint:

```text
POST /api/spike/workflow-roundtrip
Authorization: Bearer <Microsoft Graph delegated token>
```

The endpoint should:

1. Resolve the SharePoint site.
2. Resolve the workflow list by display name.
3. Create a test item.
4. Read the created item.
5. Update `memoText` and `promiseStatus`.
6. Read the updated item.
7. Delete the test item.
8. Return cleanup status.

## 6. Expected Result

## 7. Frontend UX

Add to `/spike/auth`:

```text
Call Workflow list round-trip
```

This keeps C8 as a spike route and avoids mixing workflow metadata UI
into the production Workbench before the server metadata surface is
proven.
