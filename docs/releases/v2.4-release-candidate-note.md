## 1. Release Scope

v2.4 promotes the AGING APP from browser-local persistence to the first
SharePoint-backed Phase 3 workflow foundation.

Included:

- C5 Entra Auth Spike.
- C6 Server Snapshot Upload.
- C7 Server Snapshot Read-back.
- C8 Workflow Metadata List Integration.
- C9 Owner/Memo Lite UI.
- C9.1 Owner/Memo Row Indicators.
- C10 Promise Tracking Lite.
- C11 Email Draft Lite.

## 2. Product Definition

v2.4 keeps the product identity established in v2.3:

```text
Calm accounting workbench, not alert dashboard.
```

The new value is not automation for its own sake. The new value is that
collector workflow context can move from one browser session into a
SharePoint-backed workspace.

## 4. Included Feature Summary

### 4.1 C5 Auth

Microsoft Entra delegated sign-in is proven.

API token validation reaches:

```text
GET /api/spike/whoami
```

### 4.2 C6 Snapshot Upload

Browser-local C3 snapshots can be uploaded through `aging-api` to the
SharePoint document folder.

### 4.3 C7 Snapshot Read-back

The latest SharePoint snapshot can be read back through `aging-api`,
stored into browser localStorage, and used by `/aging`.

### 4.4 C8 Workflow Metadata List

`aging-api` can create, read, update, and delete SharePoint list items in
`PE Aging&Statement Workflow`.

The key implementation rule is SharePoint display-name to internal-name
mapping. The API must not assume display names are the write field names.

### 4.5 C9 Owner/Memo

Expanded party rows can save and load:

```text
Owner
Memo
```

### 4.6 C9.1 Row Indicators

Collapsed party rows can show:

```text
Owner: ...
Memo
```

### 4.7 C10 Promise Tracking

Expanded party rows can save and load:

```text
Promise Date
Promise Amount
Promise Status
```

Collapsed party rows can show a Promise badge.

### 4.8 C11 Email Draft

Expanded party rows generate a safe email draft from party, balance,
memo, and promise context.

Actions:

```text
Copy Draft
Open Email Draft
```

C11 does not send email automatically.

## 5. Accounting Safety Rules Maintained

v2.4 preserves:

- USD and CAD separation.
- AR/AP direction separation.
- Party/currency/direction workflow keys.
- NOT_IN_ERP_EXTRACT workflow separation.
- Browser-local fallback.
- No automatic email sending.

## 7. Known Boundaries

v2.4 does not include:

- Automatic email sending.
- Microsoft Graph Mail.Send permission.
- Recipient/contact lookup.
- Multi-user conflict resolution.
- Promise history.
- Owner picker from tenant users.
- Workflow metadata filtering.
- SharePoint pagination beyond the current lite path.
- K1 alias normalization implementation.
