## Quote-style brittleness fix (Step 6 + Step 8 tests)

While converting wrappers to delegate, I changed string literals from
JSX double-quote (`<TraceField label="Source File"`) to JS object
property single-quote (`{ label: 'Source File' }`). Step 6/8 tests
were checking the exact `"Source File"` substring which broke.

The right fix is making the tests quote-agnostic, since the
*semantic* invariant is "the label string exists in source", not
"the quote style is double". Patched 5 + 4 = 9 checks across the two
test files to accept either quote style. This is more robust than
forcing a particular code style.

## Methodology note

Step 10 illustrates a useful refactor pattern:

1. **Pin existing behavior with invariants BEFORE refactoring**
   (here: UI labels, rawRow suppression, jump button text)
2. **Add new architecture alongside old** (TracePanel + Model)
3. **Convert call sites one at a time** with regression check after each
4. **Patch test brittleness as it surfaces** — sign of test
   over-specification, not test value
5. **Pin the new architecture** with new invariants + Phase 3
   anti-introduction checks

This is what allowed Step 10 to absorb in 1 round without behavior
drift. Same discipline as v2.1.1 partyName fix and Step 7 reviewer
corrections: invariants are the contract, not the prose.
