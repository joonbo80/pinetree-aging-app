### Spec compliance

| Spec rule | Implementation |
|-----------|---------------|
| Group rows touching this party | Selector pre-filters; component just renders |
| Group rows expand to show member tx ids | Click chevron toggles expandedKey state |
| Empty state: "No duplicate groups" | Placeholder branch when length === 0 |
| Tab visible even when 0 groups | Placeholder branch keeps tab in tablist |
| Per-currency totals | Two separate accumulators (D5) |

### Filter bar (3 controls)

| Field | Options |
|---|---|
| Currency | All / USD / CAD |
| Search | identityKey + member ourRefNo / invoiceNo / crdrNo / vendorInvoiceNo |
| Sort | Impact desc (default) / Impact asc / Members desc / Identity key |

Default sort = impact desc, matching Review Queue's duplicate page
ergonomics: biggest financial exposure first.

### Member rows (expand panel)

Each group expands inline to show member transactions in a sub-table:

- Type (INVOICE / CRDR / AP)
- Direction badge (recv / pay / settled)
- Amount (signed)
- Date (uses displayDate helper from Step 6)
- Reference (uses pickReference from Step 6)
- Source file:row
- "-> View tx" button -> cross-tab focus

Cross-tab focus reuses Step 7 mechanism unchanged: setFocusedTxId,
switch tab, OPEN-filter exception, scrollIntoView, 1.8s flash.

### Defensive: missing member tx

If a group's transactionIds[] contains an id that is NOT in the
party's transactions (theoretical edge case — baseline has 0 such
cases), render a yellow warning row instead of crashing. Prevents the
class of silent NPE that would never surface in baseline tests.

## Methodology note

Step 9 confirms the pattern: when a step is pure assembly on top of
established mechanisms, no micro-spec is needed and 1 round absorbs
cleanly. Compare:

| Step | Type | Rounds |
|------|------|--------|
| 6    | New mechanism (OPEN filter, sort dropdown) | 2-3 (would have been 1 with micro-spec) |
| 7    | New mechanism (cross-tab focus, broad/strict labels) | 1 (with micro-spec) |
| 8    | Assembly (reuses 6+7 patterns) | 1 (no micro-spec) |
| 9    | Assembly (reuses 6+7 patterns) | 1 expected |

The discipline: micro-spec when introducing genuinely new mechanism;
straight to code when reusing established patterns. This avoids both
overhead-of-bureaucracy and surprise-of-untested-design.

Steps 10-13 should all be 1-round absorbs.
