## Purpose

F1 is the hands-on gate for v2.3 release candidate readiness.

This walkthrough verifies that the absorbed v2.3 surface works as a real
accounting user flow, not only as isolated invariants. It focuses on the
Statement Collection Workbench, the C4 polish pass, and the C1 preflight
rules that prevent silent browser-session failures.

## Findings Template

Use this format for any issue:

```text
[CATEGORY] short title
Where:        scenario / step / URL or component
What I saw:   one or two sentences
What I expected: expected behavior
Repro:        steps
Console:      related console output, if any
```

Categories:

- BLOCKER: must fix before v2.3 release candidate.
- POLISH: acceptable for RC only if explicitly accepted.
- DEFER: track after v2.3.
